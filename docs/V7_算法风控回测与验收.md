# V7 算法风控回测与验收 / Algorithms, Risk, Backtest and Acceptance

## Training Pipeline / 训练流程

V7 baseline training 位于 `src/quantagent/training/v7_experiment.py`。默认使用 Ridge，支持 ElasticNet，Deep Alpha 保持 optional disabled，除非接入真实训练代码。

训练命令：

```powershell
quantagent train-alpha-v7 --dataset data/v7/training_dataset.parquet --output-dir artifacts/v7_alpha
```

训练保存：

- model artifact：`model_coefficients.json`
- feature schema：`feature_schema.json`
- label schema：`label_schema.json`
- training config：`training_config.json`
- metrics：`metrics.json`
- data quality report：`data_quality_report.json`
- acceptance report：`acceptance_report.json`
- git commit hash when available。

Validation 使用 purged walk-forward CV with embargo，按 horizon 生成 out-of-sample predictions 和 RankIC / net return / drawdown metrics。

## Acceptance Gates / 验收门槛

Model 不能标记 production-ready，除非以下 gate 全部通过：

- zero PIT violations。
- minimum training rows per horizon。
- minimum symbol coverage。
- minimum date coverage。
- positive out-of-sample RankIC stability。
- turnover-adjusted net return after cost。
- max drawdown below configured threshold。
- no single factor dominates unrealistically。
- performance survives at least one adverse market regime。
- paper trading report exists before live readiness can pass。
- not mock-data-only。

`v7-live-readiness-report` 只生成 readiness report，不会开启 live trading：

```powershell
quantagent v7-live-readiness-report --metrics artifacts/v7_alpha/metrics.json --paper-report reports/v7/paper_trade_report.json
```

## Backtest Boundary / 回测边界

`backtest/full_pipeline_backtester.py` 保留为 lightweight full-pipeline PIT scaffold，用于按 `as_of_date` 重放 research stages 和 PIT audit。它不是 production-grade execution simulator。

Production-grade A-share execution simulation 位于：

```text
src/quantagent/backtest/ashare_execution_simulator.py
```

它消费同一份 `target_weights`，通过 `OrderManager` 转换 order intents，再交给 `VirtualBroker` dry-run simulation。

```powershell
quantagent walk-forward-backtest-v7 --target-weights reports/v7/target_weights.csv --market-panel data/v7/market_panel.parquet
quantagent paper-trade-v7 --target-weights reports/v7/target_weights.csv --market-panel data/v7/market_panel.parquet
```

## A-share Simulation / A 股执行模拟

Execution simulation 覆盖：

- T+1 sell restriction。
- limit-up buy block。
- limit-down sell block。
- suspension block。
- ST buy block / reduction。
- lot size 100。
- volume participation cap。
- slippage curve。
- transaction cost through `VirtualBroker` cost model。
- cash handling。
- partial fills。
- failed order audit。

所有 backtest / paper trading 都必须保持 `target_weights -> OrderManager -> VirtualBroker`，不得让 Agent 或 Optimizer 直接生成 broker orders。

## Risk Boundary / 风控边界

Risk Gate 和 Kill Switch 必须先于 live submit path。任何 live order path 必须显式配置 `live_trading_enabled=true`、`dry_run=false`，并通过 risk gate、kill switch、execution constraint simulation、reconciliation 和 audit replay。默认配置永远保持 dry-run / virtual-broker only。
