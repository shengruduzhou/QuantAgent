# V6 模型训练与验证 / Model Training and Validation

## 目标 / Goal
把统一 `MultiTowerModel` 生产化为 `V6ModelSystem`，支持 multi-horizon alpha、direction、quantile、confidence、risk_score、factor_gate 和 conformal inference。

## 架构 / Architecture
`models/multitower.py` 是 V4/V5/V6 共用实现，`v4_multitower.py` 和 `v5_multitower.py` 只保留 compatibility wrapper；`models/v6_model_system.py` 负责 inference facade；`training/v6_dataset.py` 构造 multi-horizon labels；`training/train_v6_multitower.py` 提供 CPU smoke training；`training/model_registry.py` 保存 registry metadata。

## 数据流 / Data Flow
PIT features 进入 dataset builder，训练或 dry-run 后写入 model registry；inference 输出 `V6ModelOutput`，再由 conformal calibrator 生成 q_low / q_high / conformal_confidence。

## 关键模块 / Key Modules
`v6_losses.py` 包含 rank loss、pinball loss 和 multi-horizon consistency loss；`validation_report.py` 输出 walk-forward、purged CV、embargo 和 no-lookahead checks。

## CLI 使用方式 / CLI
```powershell
quantagent train-v6 --config configs/v6.default.yaml --dry-run
quantagent infer-v6 --config configs/v6.default.yaml --date 2026-03-31
quantagent validate-v6 --config configs/v6.default.yaml
```

## 配置方式 / Config
模型参数位于 `configs/model.v6.yaml` 和 `configs/v6.default.yaml` 的 `model`、`training` 节。

## 安全边界 / Safety
训练测试可 skip heavy torch，但 non-training V6 pipeline smoke 必须通过；模型不会输出 order。

## 测试方式 / Testing
`tests/test_v6_conformal_inference.py` 验证 conformal confidence 被 inference 消费。

## 验收标准 / Acceptance
输出包含 1d / 5d / 20d alpha、direction logits、quantile interval、confidence、risk_score、factor_gate、model_version 和 feature_version。
