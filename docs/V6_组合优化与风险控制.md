# V6 组合优化与风险控制 / Portfolio and Risk

## 目标 / Goal
把 model alpha、conformal confidence 和 agent views 转成 posterior alpha，并通过 regime-aware optimizer 与 RiskGate 输出受控 target_weights。

## 架构 / Architecture
`portfolio/v6_portfolio_service.py` 调用 AgentRouter、Bayesian arbitration、`solve_v5_portfolio` 和 `RiskGate`。

## 数据流 / Data Flow
V6ModelOutput 的 alpha 先按 confidence、conformal_confidence 和 risk_score 降权，再与 AgentView posterior 融合，optimizer 输出 target_weights，RiskGate 检查权重和订单两层风险。

## 关键模块 / Key Modules
`risk/risk_gate.py`、`risk/kill_switch.py`、`risk/risk_limits.py`、`quant_math/regime_aware_optimizer.py`。

## CLI 使用方式 / CLI
```powershell
quantagent build-portfolio-v6 --config configs/v6.default.yaml
```

## 配置方式 / Config
`configs/risk_limits.v6.yaml` 和 `configs/v6.default.yaml` 的 `portfolio`、`execution` 节控制约束。

## 安全边界 / Safety
RiskGate 检查 max_name_weight、max_sector_weight、max_turnover、停牌、涨跌停、ST、cash、leverage、uncertainty、data_quality、model_drift 和 kill_switch。

## 测试方式 / Testing
`tests/test_v6_risk_gate.py` 和 `tests/test_v6_kill_switch.py` 覆盖拒绝路径。

## 验收标准 / Acceptance
非法 target/order 被拒绝且 rejected_symbols 可解释；kill switch 后不能 submit 新订单。
