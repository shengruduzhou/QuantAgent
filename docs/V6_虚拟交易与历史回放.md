# V6 虚拟交易与历史回放 / Virtual Trading and Replay

## 目标 / Goal
支持 VirtualBroker、HistoricalLiveReplay 和 PaperTrading，不依赖真实券商、真实账户或真实 QMT。

## 架构 / Architecture
`execution/virtual_broker.py` 模拟账户、现金、持仓、部分成交、滑点和费用；`replay/historical_live_replay.py` 按 scenario 驱动每日 pipeline。

## 数据流 / Data Flow
target_weights 进入 OrderManager，生成 order intents；RiskGate 通过后提交给 VirtualBroker；成交、取消、拒单写 audit；Reconciliation 每日核对。

## 关键模块 / Key Modules
`virtual_broker.py`、`fill_simulator.py`、`cost_model.py`、`position_ledger.py`、`reconciliation.py`、`scenario_registry.py`。

## CLI 使用方式 / CLI
```powershell
quantagent replay-v6 --config configs/v6.default.yaml --scenario mock_recent_replay
quantagent paper-trade-v6 --config configs/v6.default.yaml --dry-run
```

## 配置方式 / Config
`configs/replay_scenarios.v6.yaml` 支持多年份、多日期、多 universe、多账户规模和 mock / real_external data_mode。

## 安全边界 / Safety
当前 live trading 仅表示 virtual live trading、historical live replay 和 paper trading。真实券商 adapter 不存在且默认禁用。

## 测试方式 / Testing
`tests/test_v6_virtual_broker.py`、`tests/test_v6_replay_scenarios.py` 和 `tests/test_v6_cli_smoke.py` 覆盖虚拟交易和回放。

## 验收标准 / Acceptance
VirtualBroker 不连接真实账户；涨跌停、停牌、T+1、费用和 slippage 在虚拟层可审计。
