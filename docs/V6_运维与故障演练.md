# V6 运维与故障演练 / Operations and Failure Drills

## 目标 / Goal
让 provider failure、model drift、data quality warning、risk rejection 和 audit failure 都有明确降级路径。

## 架构 / Architecture
provider adapter 负责 fallback；RiskGate 与 KillSwitch 负责 hard stop；ValidationReport 与 readiness report 记录状态。

## 数据流 / Data Flow
异常先进入 warning / diagnostics，再由 service 汇总到 report；严重异常触发 kill switch，订单路径停止。

## 关键模块 / Key Modules
`risk/kill_switch.py`、`risk/risk_gate.py`、`services/v6_pipeline_service.py`、`execution/audit_replay.py`。

## CLI 使用方式 / CLI
```powershell
quantagent validate-v6 --config configs/v6.default.yaml
quantagent generate-v6-report --config configs/v6.default.yaml --output-dir reports/v6
```

## 配置方式 / Config
`safety.require_kill_switch`、`safety.require_reconciliation`、`data.min_data_quality_score` 和 `risk_limits` 控制故障阈值。

## 安全边界 / Safety
故障演练只能作用于 mock、historical replay 或 paper trading，不允许真实账户下单。

## 测试方式 / Testing
使用 pytest 的 failure-mode tests 覆盖 kill switch、provider mock fallback、RiskGate rejection 和 audit replay。

## 验收标准 / Acceptance
关键故障不会静默通过；report 明确 warning、remaining gaps 和是否触发 kill switch。
