# V6 生产可信度与审计 / Production Trust

## 目标 / Goal
所有策略输出必须可审计、可重放、可解释、可追责。

## 架构 / Architecture
AuditLogger 写 append-only JSONL；AuditReplay 重放事件；Reconciliation 对 target、broker position、ledger、cash、fills 和 order states 做核对。

## 数据流 / Data Flow
provider quality warning、agent evidence、portfolio diagnostics、risk rejection、order submit、fill 和 reconciliation report 都进入审计链。

## 关键模块 / Key Modules
`execution/audit.py`、`execution/audit_replay.py`、`execution/reconciliation.py`、`training/validation_report.py`。

## CLI 使用方式 / CLI
```powershell
quantagent audit-replay-v6 --audit-path logs/v6/virtual_broker_audit.jsonl
quantagent generate-v6-report --config configs/v6.default.yaml
```

## 配置方式 / Config
`execution.audit_log_dir` 控制 audit 输出目录，`reporting.output_dir` 控制 validation 和 readiness report。

## 安全边界 / Safety
audit write failure 可触发 kill switch；真实 broker 不在 V6 默认路径中。

## 测试方式 / Testing
`tests/test_v6_reconciliation.py` 和 `tests/test_v6_audit_replay.py` 验证 report 与 replay。

## 验收标准 / Acceptance
每日 reconciliation report 包含 expected_position、broker_position、difference、cash_difference、unresolved_orders、rejected_orders、fill_rate、slippage、cost、turnover。
