# V6 验收标准 / Acceptance Criteria

## 目标 / Goal
定义 V6 是否完成的最低可运行标准和 readiness score。

## 架构 / Architecture
验收覆盖 Data & Feature OS、Model & Training OS、Agent Evidence OS、Portfolio & Risk OS、Virtual Trading OS 和 Production Trust OS。

## 数据流 / Data Flow
必须跑通 data -> feature -> model -> inference -> agent -> optimizer -> risk -> virtual execution -> reconciliation -> attribution/report。

## 关键模块 / Key Modules
V6 CLI、provider adapters、V6ModelSystem、AgentCommittee、RiskGate、KillSwitch、VirtualBroker、Reconciliation、AuditReplay、ValidationReport。

## CLI 使用方式 / CLI
```powershell
python -m pytest tests/ -q
quantagent validate-v6 --config configs/v6.default.yaml
quantagent replay-v6 --config configs/v6.default.yaml --scenario 2020_covid_volatility_replay
quantagent paper-trade-v6 --config configs/v6.default.yaml --dry-run
```

## 配置方式 / Config
所有验收命令默认读取 `configs/v6.default.yaml`，不得要求真实 API key 或真实账户。

## 安全边界 / Safety
Agent 能直接生成 order、真实券商默认可用、没有 VirtualBroker、没有 RiskGate、没有 KillSwitch、没有 Reconciliation、没有 AuditReplay，任一项发生即失败。

## 测试方式 / Testing
核心测试包括 no-lookahead、provider mock、factor gate closed loop、agent reliability router、conformal inference、risk gate、kill switch、virtual broker、reconciliation、audit replay、replay scenarios、CLI smoke 和 docs exist。

## 验收标准 / Acceptance
目标 readiness score：模型设计 >= 9.3，工程落地 >= 9.0，闭环完成度 >= 9.0，生产可信度 >= 8.8，安全边界 >= 9.5；无法达到的项目必须列入 remaining gaps。
