# V7 证据摄取与交易规则 / Evidence Ingestion and Trading Rules

## 目标 / Goal

V7 evidence layer 把政策、公告、新闻、财报、订单合同、监管处罚统一成 PIT-safe frame。每条 evidence 必须可以回答：来自哪里、何时发布、何时可见、可信度多少、是否有 cross-source validation、是否会衰减、原文 hash 是什么。

## Unified Evidence Schema / 统一证据字段

所有 ingestor 输出最终会归一到 `EVIDENCE_COLUMNS`，核心字段包括：

```text
source_name / source_type / source_authority / source_reliability
url / title / body
published_at / available_at / ingested_at
symbol / company_name / affected_symbols
theme_candidates / chain_node_candidates
event_type / confidence
cross_validation_count / contradiction_count
horizon_days / decay_half_life
rumor_risk_flag / raw_hash / point_in_time_valid
```

`EvidenceStore` 按 `available_at=YYYY-MM-DD` 分区写入 Parquet 或 CSV fallback。`read_visible(as_of_date)` 只返回当日可见 evidence。

## Public Crawler / 公开网页爬虫

`src/quantagent/data/crawler/` 是新的 crawler package。它保留旧的 `PublicWebCrawler` facade，但实现拆成：

- `config.py`
- `http_client.py`
- `proxy_pool.py`
- `rate_limiter.py`
- `robots_policy.py`
- `fetcher.py`
- `parser.py`
- `canonicalize.py`
- `metrics.py`
- `firecrawl_like.py`

默认行为：

- `allow_network=false`，显式开启才联网。
- global rate limit 不超过 5 requests/second。
- per-domain rate limit 不超过 1 request/second。
- 支持 robots.txt 和 domain allowlist。
- 支持 ETag / Last-Modified。
- canonical URL 去重，content hash 去重。
- CAPTCHA / blocked page 只标记 blocked，不绕过、不代理规避、不形成 retry storm。

Proxy support 只用于 privacy、reliability 和 controlled egress。禁止绕过 CAPTCHA、login wall、paywall、robots.txt 或 explicit rate limits。

## News Quality / 新闻证据质量

`NewsIngestor` 使用 layered tagging：

1. exact keyword seed；
2. company/security dictionary match；
3. industry/theme ontology match；
4. optional embedding reranker；
5. cross-source validation。

规则：

- rumor keywords 会降低 confidence。
- single low-reliability source 且缺 cross-validation 时，`core_pool_signal_allowed=false`，confidence cap 到低位。
- news row 必须保留 `url / raw_hash / available_at / source_reliability / cross_validation_count / contradiction_count`。

## Stock Pool and Trading Rules / 股票池与交易规则

`stock_pool_gate` 在 alpha 前执行：

- core + strong 默认允许；
- satellite 只有在 confidence 达标时允许；
- watchlist、exclusion、false-association 和 no-factor-coverage 默认剔除；
- gate 为空时 fail closed，不回退 full universe。

之后的组合流程只输出 `target_weights`。交易约束模拟覆盖：

- T+1；
- limit-up no buy；
- limit-down no sell；
- suspension；
- ST；
- liquidity；
- lot size；
- volume participation cap；
- slippage and impact。

Risk Gate 与 Kill Switch 必须在任何 QMT submit path 之前运行。
