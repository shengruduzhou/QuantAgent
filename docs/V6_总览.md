# V6 总览 / Overview

## 目标 / Goal
QuantAgent V6 的目标是形成接近生产级的 AI Quant OS，覆盖 data、feature、model、agent evidence、portfolio、risk、virtual execution、reconciliation、audit replay 和 validation report。

## 架构 / Architecture
主链为 `ExternalDataProviders -> PIT FeatureStore V6 -> FactorPipeline V6 -> V6ModelSystem -> AgentCommittee -> AgentRouter -> BL posterior -> Regime-aware Optimizer -> RiskGate -> OrderManager -> VirtualBroker -> Reconciliation -> AuditReplay`。

## 数据流 / Data Flow
provider adapter 拉取或模拟行情、新闻、财务、资金流、宏观和商品数据；FeatureStore 只使用 as-of cutoff 前已知数据；模型输出 alpha / uncertainty / factor_gate；Agent 只输出 EvidenceRecord；OrderManager 生成 order intents；VirtualBroker 虚拟成交并写 audit。

## 关键模块 / Key Modules
`src/quantagent/services/v6_pipeline_service.py` 是端到端 service；`models/v6_model_system.py` 是 inference facade；`portfolio/v6_portfolio_service.py` 串接 posterior 和 optimizer；`risk/` 和 `execution/` 提供风控与虚拟执行。

## CLI 使用方式 / CLI
```powershell
quantagent validate-v6 --config configs/v6.default.yaml
quantagent replay-v6 --config configs/v6.default.yaml --scenario mock_recent_replay
quantagent paper-trade-v6 --config configs/v6.default.yaml --dry-run
```

## 配置方式 / Config
使用 `configs/v6.default.yaml` 控制 safety、data、model、training、agents、portfolio、execution 和 reporting。

## 安全边界 / Safety
V6 默认 `virtual_broker_only=true`、`dry_run=true`、`real_broker_enabled=false`。Agent 与 optimizer 都不能生成 broker order。

## 测试方式 / Testing
运行 `python -m pytest tests/ -q`。单元测试使用 mock provider，不访问外部网络。

## 验收标准 / Acceptance
V6 CLI 存在，pipeline 可端到端运行，RiskGate / KillSwitch / VirtualBroker / Reconciliation / AuditReplay 均可用，中文 V6 文档存在。
